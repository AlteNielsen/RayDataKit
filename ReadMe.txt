Ray Data Kit for Unity 使用ガイド

☆概要
Ray Data Kitは製品全域の状態の保持、保存/復元、マスターデータの管理を目的とした超小規模フレームワークです。
構造体単位でデータの管理ができる利便性と、全てのデータを一つの配列で管理することによる高速性を兼ね備えています。
状態の管理に特化しているため、float型以外の型は一切使用できません。

☆注意点
本フレームワークを使用するためには、本フォルダをAssetsフォルダーに入れた上で、作成したC#スクリプトにそれぞれ
using Ray.Data;
を書いておく必要があります。


☆データクラス定義
シーンごとなど、扱いたい範囲のデータごとに管理用のクラスを作成します。
RayDataChunkを継承したクラスを用意してください。

Load()メソッドは主にファイルからのデータ読み込みに使用します。
起動時にマネージャーに呼び出されます。
戻り値には扱いたいデータをfloat配列化したものを指定してください。

Save()とdataは後に解説します。


☆マネージャー定義と構造体登録
各データ構造体と全てのデータを管理するマネージャーを作成します。
RayAbstractDataManagerを継承したクラスを用意してください。
次に、
StructRegistry()
メソッド内で、
Register<T>
メソッドを用いて、先ほど作成したデータ構造体を全て登録してください。
これにより、起動時に各データ構造体のdataに、マスターバッファの対応する部分の参照が渡されます。


☆稼働時におけるデータの変更について
稼働時において、データの取得・変更は全てマネージャーを通して行われるようにしてください。

GetData(int selector, float[] frame)
メソッドを用いることで、各データ構造体に対応する変数全てを一括で取得できます。
第一引数はStructRegistry()内で上から何番目に登録したものを使いたいかを指定します。
enumで管理することを強く推奨いたします。
結果は第二引数にコピーされます。

SetData(int selector, float[] values)
メソッドを用いることで、各データ構造体に対応する変数全てを一括で変更できます。
第一引数はGetDataメソッドと同じ仕組みです。
第二引数の内容がマスターバッファの対応する領域にコピーされます。


☆データクラスのdataと関数について
dataには、実際にはマスターバッファの対応する領域の参照と長さが入っていますが、
使用上では通常の配列として扱うことができます。
data[n]
という形でデータを参照できます。
クラス内部でそのデータを利用した関数を定義したい時にご利用ください。
外部からは、マネージャーの
T Access<T>(int key) where T : RayDataChunk
メソッドを用いることでクラスへアクセスすることができます。
引数は、先に解説したGetData、SetDataメソッドにおける第一引数と同じ仕組みです。

注意点として、全体の同期を取るために守るべきルールが存在します。
まず、各構造体からマスターバッファのデータをdataを通して変更することはできません。
また、そのクラスに対応するデータ領域が同時に外部で変更されている可能性がある場合、そのクラスの関数を呼び出さないようにしてください。
これはクラスの持つデータ(実際にはマスターバッファに存在)と、外部で情報の同期が行われていない状態であるため、その状態で関数を呼び出すと古いデータを元にした結果になってしまい、外部における最新データとの食い違いが起こってしまうためです。

☆Saveについて
各データ構造体にはSave()メソッドが存在します。
中身は完全に自由で、Jsonやcsvなど、皆様思い思いの形式で情報を保存することができます。
マネージャーに存在するSave()メソッドを呼び出すことで、全てのデータ構造体のSave()を動かすことができます。ご活用ください。

☆ScriptableObject対応版について
RayAbstractSOManagerとRaySOChunkを追加しました。
float → ScriptableObject
RayAbstractDataManager → RayAbstractSOManager
RayDataChunk → RaySOChunk
という対応関係になっています。
使い方の変更点としましては、
Save、GetData、SetDataメソッドが存在しません。
それ以外は変更ありません。
